「Luogu4158」[SCOI2009]粉刷匠

problem

题目描述

windywindywindy有 NNN 条木板需要被粉刷。 每条木板被分为 MMM 个格子。 每个格子要被刷成红色或蓝色。

windywindywindy每次粉刷，只能选择一条木板上一段连续的格子，然后涂上一种颜色。 每个格子最多只能被粉刷一次。

如果windywindywindy只能粉刷 TTT 次，他最多能正确粉刷多少格子？

一个格子如果未被粉刷或者被粉刷错颜色，就算错误粉刷。

输入输出格式

输入格式：

第一行包含三个整数，NNN MMM TTT。

接下来有NNN行，每行一个长度为MMM的字符串，'0’表示红色，'1’表示蓝色。

输出格式：

包含一个整数，最多能正确粉刷的格子数。

输入样例#1：

输出样例#1：

说明

30%30\%30%的数据，满足 1≤N,M≤101 \le N,M \le 101≤N,M≤10 ； 0≤T≤1000 \le T \le 1000≤T≤100 。

100%100\%100%的数据，满足 1≤N,M<=501 \le N,M <= 501≤N,M<=50 ； 0≤T≤25000 \le T \le 25000≤T≤2500 。

Solution

蒟蒻看到这题想了nnn多种完全不正确的处理方法，如果是在考场上估计已经光速凉凉了QAQQAQQAQ

对于每一行，我们可以把它分成若干个颜色不同的连续段（对应若干次颜色不同的粉刷），从左到右考虑

注意到题目：

一个格子如果未被粉刷或者被粉刷错颜色，就算错误粉刷。

意思就是不刷白不刷

如果是颜色不对，多刷这一格不会比不刷差，就不需要考虑不刷的情况

所以我们有：

状态：设dpi,j,k,0/1dp_{i,j,k,0/1}dpi,j,k,0/1​表示当前枚举到第iii行，第jjj个格子，已经涂了kkk次（分成了kkk段），当前格子涂的颜色是0/10/10/1

方程：

dpi,j,k,x=max(dpi,j−1,k,x,dpi,j−1,k−1,x xor 1)+[x=colori,j]dp_{i,j,k,x}=max(dp_{i,j-1,k,x},dp_{i,j-1,k-1,x\space xor\space 1})+[x=color_{i,j}] dpi,j,k,x​=max(dpi,j−1,k,x​,dpi,j−1,k−1,x xor 1​)+[x=colori,j​]

(方括号是艾弗森括号，当其中的条件为真时值为111，否则为000)

这个转移应该很好理解吧

处理完每一行之后，max(dpi,m,k,0,dpi,m,k,1)max(dp_{i,m,k,0},dp_{i,m,k,1})max(dpi,m,k,0​,dpi,m,k,1​)即为第iii行分配kkk次粉刷次数所能正确粉刷的最大格子数

这样，每一行作为一组，原问题就转化为一个分组背包问题，在此不再赘述
